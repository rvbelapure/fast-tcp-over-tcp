CSE 6230, Fall 2013: Lab 3, Th Sep 19: CUDA

  • This page: http://j.mp/gtcse6230fa13lab3
  • Info on the Jinx cluster: http://support.cc.gatech.edu/facilities/instructional-labs/jinx-cluster
  • Info on CUDA: http://docs.nvidia.com/cuda/cuda-c-programming-guide/index.html

In this lab, you will learn to write simple CUDA programs. The lab will consist of two parts. The first part will be done in class and will involve writing a simple SAXPY (y = a * x + y) function. The second part of the lab can be done at home, and will involve writing an optimized matrix transpose function.

You may if you wish work in teams of two. To simplify our grading of your assignments, each person should submit his/her own assignment; however, all team members may submit identical code. Be sure to indicate with whom you worked by creating a README file as part of your submission. (See below for details.)

Part 0: Getting started

We've written some skeleton code for you to use for this lab. To get it, use the same fork-checkout procedure from lab1. The repo you want is gtcse6230fa13/lab3.

If it worked, you'll have a lab3 subdirectory containing 5 files.

Part 1: SAXPY function in CUDA

Open and read the saxpy.cu file containing the skeleton code for the SAXPY function. It performs the element-by-element operation y = a * x + y for two equally sized input arrays x and y and the constant a.

All of the host code needed for this function has been provided, including the code for allocating memory and copying data to and from the device. All you have to do is fill in the code for the GPU kernel function, __global__ void saxpy

The total number of usual threads has already been given and should not be changed. Write your kernel function in such a way that it can handle arbitrarily sized arrays using only those threads.

When you are ready to compile, type nvcc saxpy.cu -o saxpy which produces saxpy. You can run this binary using the provided cuda.pbs file.

Once you've got something working, use the usual add-commit-push steps to save this version of your repo, and "share" it with us (gtcse6230fa13) on Bitbucket.

Part 2: Matrix transpose [Due: Sep 19, 2013 before class]

The repo we've provided includes another file, transpose.cu which contains the skeleton code for transposing a matrix on the GPU. This time, only the code for creating the input matrix on the host has been given. This means that you'll have to write the code needed for allocating memory for the matrix on the GPU, as well as communication to and fro. More specifically, edit the functions void gpuTranspose and __global__ void matTrans without changing any of the provided code.

Although you have been given freedom to write your own code for this part, you CANNOT change the data structure of the matrix on the host. You may, however, pad the array to ensure that each row starts from an aligned memory address.

Your performance target for a guaranteed "A" this week is 10 billion elements per second. Recall that this is a sufficient condition, but not necessary, to receive an "A".
